## Priority Issues

### P1: Telegram env-var graceful degradation
`Contact.tsx` handleSubmit: if `NEXT_PUBLIC_TG_TOKEN`/`NEXT_PUBLIC_TG_CHAT_ID` are unset, every submission instantly errors with the generic "Не отправилось" message — indistinguishable from a real network failure to the user.

### P2: `.contactAction` missing focus-visible — FIXED in this pass
Added `:focus-visible` rule mirroring `:hover` plus an outline, consistent with other interactive elements site-wide.

### P3: `noValidate` + `required` is a no-op
`<form noValidate>` disables native browser validation, but `required` attributes remain with no client-side replacement. Either remove `noValidate` or add real validation.

### P4 (minor): Hero animation timeline length
Minor tuning opportunity in hero reveal sequencing; not urgent.
